## 1. Strategy Overview

SEO is Corredor's #1 long-term acquisition channel (lowest CAC, compounding returns, brand authority). The strategy has two phases:

- **Pre-launch (W20–39):** Build topical authority before GA. Get content indexed and ranking so organic traffic converts on launch day.
- **Post-launch (W40+):** Scale with programmatic SEO using per-neighborhood market data pages and tenant property listing pages.

**Domain authority starting point:** 0 (new domain). Target: DA 25+ by month 12.

## 5. Technical SEO Requirements

### Next.js App (apps/site)

#### Structured Data (JSON-LD)

**Homepage — LocalBusiness:**
```json
{
  "@context": "https://schema.org",
  "@type": "SoftwareApplication",
  "name": "Corredor CRM",
  "applicationCategory": "BusinessApplication",
  "operatingSystem": "Web",
  "offers": {
    "@type": "Offer",
    "price": "12",
    "priceCurrency": "USD"
  },
  "description": "CRM inmobiliario para agencias y corredores en Argentina",
  "inLanguage": "es-AR"
}
```

**Property Listing Pages (per-tenant) — RealEstateListing:**
```json
{
  "@context": "https://schema.org",
  "@type": "RealEstateListing",
  "name": "Departamento 3 ambientes en Palermo",
  "url": "https://agencia.corredor.app/listing/123",
  "datePosted": "2026-04-20",
  "image": "...",
  "address": {
    "@type": "PostalAddress",
    "addressLocality": "Palermo",
    "addressRegion": "Buenos Aires",
    "addressCountry": "AR"
  }
}
```

#### Core Web Vitals Targets

| Metric | Target | Current (new domain) |
|---|---|---|
| LCP (Largest Contentful Paint) | < 2.5s | TBD |
| FID / INP (Interaction to Next Paint) | < 200ms | TBD |
| CLS (Cumulative Layout Shift) | < 0.1 | TBD |
| TTFB (Time to First Byte) | < 600ms | TBD |

**Required implementation checklist:**
- [ ] Image optimization: Next.js `<Image>` component everywhere (no raw `<img>`)
- [ ] Font optimization: `next/font` with `display: swap`
- [ ] Static generation (SSG) for blog posts and marketing pages
- [ ] ISR (Incremental Static Regeneration) for per-tenant property pages (revalidate: 3600)
- [ ] Preconnect hints for CDN and portal image domains
- [ ] Critical CSS inlined; non-critical deferred

#### Sitemap Generation

Two sitemap types needed:

1. **Marketing site sitemap** (`/sitemap.xml`):
   - Homepage, pricing, features, comparison pages, blog posts
   - Updated on every deployment
   - Target: <500 URLs total

2. **Per-tenant property sitemap** (`/[tenant-slug]/sitemap.xml`):
   - One sitemap per tenant website
   - Lists all published property pages for that tenant
   - Generated dynamically via ISR or server-side
   - Submit to Google Search Console per tenant on activation

#### hreflang & Language Tags

- Primary: `es-AR` (Argentina Spanish)
- Do NOT add multilingual support pre-launch — complexity not worth it

#### Meta Tags Template

```html
<title>Software Inmobiliario Argentina | CRM Inmobiliario — Corredor</title>
<meta name="description" content="Gestión de propiedades, CRM y publicación en portales para inmobiliarias argentinas. Probalo gratis 14 días." />
<meta property="og:title" content="..." />
<meta property="og:description" content="..." />
<meta property="og:image" content="/og-image.jpg" /> <!-- 1200×630px -->
<link rel="canonical" href="https://corredor.app/..." />
```

#### Robots & Indexing

- Block `/app/*` (authenticated SaaS app) from indexing
- Block `/api/*` from indexing
- Allow all `/blog/*`, `/features/*`, `/pricing`, `/[tenant-slug]/*`

## 6. Programmatic SEO (Post-GA)

Once GA ships and tenant websites exist at scale, generate:

### Neighborhood Market Pages (`/mercado/[barrio]`)

One page per major neighborhood with:
- Average price per m² for sale + rent
- YoY change
- Number of active listings on Corredor in that barrio
- "Agencias en [barrio] que usan Corredor" — social proof

**Target volume:** 50 neighborhood pages across CABA + 10 major cities (Córdoba, Rosario, Mendoza, Mar del Plata, etc.)

**Expected traffic at month 12:** 5,000–15,000 organic visits/month from neighborhood pages alone

### Tenant Website SEO

Each tenant website on `[slug].corredor.app` gets:
- Auto-generated meta tags per listing (title, description, OG image)
- RealEstateListing structured data per property
- Sitemap auto-submitted to Search Console on agency activation
- `rel="canonical"` to prevent duplicate content with portal listings

## 8. Search Console & Analytics Setup

Pre-launch checklist:
- [ ] Verify corredor.app in Google Search Console (primary domain)
- [ ] Submit sitemap.xml to GSC
- [ ] Configure GA4 with conversion events: trial_signup, waitlist_join, upgrade
- [ ] Set up GSC performance tracking per keyword cluster weekly
- [ ] Configure rank tracking (Ahrefs or Semrush — budget $99–199/mo)
- [ ] Set up Core Web Vitals monitoring (GSC + PageSpeed Insights CI check)

Post-GA additions:
- [ ] Verify `[tenant].corredor.app` subdomains for top beta agencies
- [ ] Set up property sitemap auto-submission pipeline
- [ ] Monthly keyword rank review against master list above
